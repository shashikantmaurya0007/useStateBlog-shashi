Blog on UseState

Before understanding useState functionality of React .lets understand Why React included this functionality? . First of all you should know React is declarative!
Wait if you are wondering what is declarative ?

React is declarative because you write the code that you want and React is in charge of taking our declared code and performing all of the Javascript steps to get you to your desired result.
If you are thinking that you still write some logic in React .you are right no component till now is complete declarative.
Lets understand the problem of vanilla-js which React is solving through useState
Consider a problem where you have given a variable num which exist in global scope and the code somewhat looks like stated below.
In the above code you have created three h1 elements with “id” one ,two ,three respectively. And one button with onclick event handler calling buttonClicked()
Let me run you through Javascript code
you have created variable “var num=1 “ in global scope and then getting the element h1 by selecting using getElementById and trying to change the innerText with "num1" value. If you see the output initially you are getting “1” which is the value of global scope variable num1.
The catch comes here when you click on the button and buttonClicked()function is called where, You are incrementing the value of "num1" and trying to console the value of "num1". In console you are getting the incremented value of “num1” where as h1.innerText is not getting the current state of num1.
The reason behind this is once the code is run it do not render again unless you explicitly put the logic in function and call that function once the state is changed to display the changes.
Don’t you feel it is more tedious work for the programer. HERE REACT COMES LIKE SUPERHERO TO SOLVE YOUR PROBLEM.
This functionality react will be achieving with useState().

useState is react component.so lets understand useState functionality.

So basically useState() require the initial state and it will be returning array with two value one variable with current state value and one function to change that value.
If the above statements seems confusing to you it would be much clear when you see the below code.
const[num,setNum]=useState(1);
so in the above code "1 " is the initial value and num is the variable which is assiged with initial value. and setNum is the function which updates the value of num.
so WhenButton is clicked setNum function *which is provided by useState* is executed and there you are incrementing the value of num1
Everytime setNum is called it generate a new state for num and React beautifully render and replace num used in code with current state. which saves lots of energy and brain of programmer.
" you can understand useState as whenever state changes React please react on it"
